SQL Table Management Prep NotesThis document covers the creation, modification, and cloning of tables in SQL (specifically for MySQL).1. The CREATE TABLE StatementThe CREATE TABLE statement is used to create a new table in a database.Basic SyntaxSQLCREATE TABLE table_name (
  column1 datatype,
  column2 datatype,
  ...
  columnN datatype
);
Example: Basic Table CreationTo create a CUSTOMERS table with basic attributes:SQLCREATE TABLE customers (
    ID INT NOT NULL,
    Name VARCHAR(20) NOT NULL,
    age INT NOT NULL,
    address CHAR(25),
    salary DECIMAL(18,2)
);
2. Defining a Primary KeyA Primary Key uniquely identifies each record in a table. You can add it during creation or by modifying an existing table.During CreationSQLCREATE TABLE customers1 (
    id INT NOT NULL PRIMARY KEY,
    name VARCHAR(20) NOT NULL,
    age INT NOT NULL,
    address CHAR(25),
    salary DECIMAL(18, 2)
);
Modifying an Existing TableIf the table already exists but lacks a Primary Key:SQLALTER TABLE customers
MODIFY ID INT NOT NULL PRIMARY KEY;
3. Safe Creation: IF NOT EXISTSUse this clause to prevent errors if you attempt to create a table that already exists.SyntaxSQLCREATE TABLE IF NOT EXISTS table_name (
   column1 datatype,
   ...
   PRIMARY KEY (one or more columns)
);
Verification: If the table exists, MySQL will issue a Note instead of an Error.SQLSHOW WARNINGS;
-- Output: Note | 1050 | Table 'customers' already exists
4. Creating a Table Using Another TableYou can clone a table's structure and data using the AS SELECT statement.SyntaxSQLCREATE TABLE NEW_TABLE_NAME AS
SELECT column1, column2, ...
FROM EXISTING_TABLE_NAME
WHERE Condition;
ExamplesFull Clone:SQLCREATE TABLE new_customers AS SELECT * FROM customers1;
Partial Clone (Specific Columns):SQLCREATE TABLE SALARY AS
SELECT ID, SALARY
FROM CUSTOMERS;
5. Metadata and Inspection CommandsUse these commands in your terminal to verify your database structure:CommandPurposeSHOW TABLES;Lists all tables in the current database.DESCRIBE table_name;Shows column names, types, and constraints (Keys, Nulls).DESC table_name;Shortened version of DESCRIBE.6. Important Best PracticesMeaningful Names: Names should be descriptive (e.g., FIRST_NAME instead of F_N).Correct Data Types: * Use DECIMAL for money/currency.Use VARCHAR for variable text (like names).Use INT for whole numbers (like IDs).Unique Identification: Every table should ideally have a PRIMARY KEY.Case Sensitivity: While SQL keywords are case-insensitive, table names can be case-sensitive depending on your operating system (Mac/Linux usually are).
